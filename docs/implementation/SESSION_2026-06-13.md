**Payment Domain — Design Decisions:**
1. **Idempotency Key** — prevents duplicate payments on retry
2. **Outbox Pattern** — all payment events critical (guaranteed delivery)
3. **Soft References** — enrollment_id as UUID (no FK)
4. **Webhook Stubs** — signature verification ready for production implementation
5. **Money Value Object** — encapsulates amount + currency logic

**Integration Flow:**
```
Payment → PaymentSucceeded (Outbox) → Enrollment → EnrollmentActivated
Payment → PaymentFailed (Outbox) → Enrollment → EnrollmentSuspended
Payment → RefundIssued (Outbox) → Enrollment → EnrollmentDropped
```
